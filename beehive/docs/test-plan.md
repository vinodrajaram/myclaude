Generated by /plan-eng-review on 2026-03-20

## Affected Pages / Routes
- `/` — Dashboard / upcoming calls (calendar view)
- `/record` — Pre-call screen, recording controls
- `/calls/:id` — Summary, action items, SSE listener for in-progress calls
- `/history` — Searchable call log
- `/api/auth/zoom` — Zoom OAuth initiate
- `/api/auth/zoom/callback` — Zoom OAuth callback + token storage
- `/api/zoom/webhook` — Zoom recording.completed webhook receiver
- `/api/calls` — List calls (paginated)
- `/api/calls/:id` — Get call detail
- `/api/calls/:id/stream` — SSE endpoint for summary-ready event
- `/api/action-items/:id` — Update/delete action items
- `/api/calls/:id/email` — Draft follow-up email
- `/api/calendar/upcoming` — Proxy to Microsoft Graph API

## Key Interactions to Verify

### Onboarding
- First launch shows consent dialog before any recording is attempted
- Zoom OAuth connect completes in < 2 minutes and shows connected state
- App is usable without calendar connected (manual call name entry works)

### Recording Pipeline (core flow)
- Banker starts a call in Zoom; app shows "waiting for recording" state
- Zoom sends `recording.completed` webhook → backend verifies HMAC signature
- Backend fetches VTT from Zoom; VTT contains speaker labels
- VTT is parsed, timestamps stripped, speakers preserved
- Clean text sent to Gemini Vertex AI with structured response schema
- Gemini returns valid JSON with summary + action items
- Summary displayed within 90 seconds of call ending
- Progress states shown: Fetching → Transcribing → Summarizing → Done
- SSE connection delivers summary-ready event to open browser tab

### Action Items
- Action items extracted from transcript appear in correct order
- Banker can edit description, owner, and due date inline
- Banker can delete an action item
- Banker can mark action item complete (checkbox)
- Uncertain-owner items flagged with visual indicator

### Follow-up Email
- "Draft Follow-up Email" CTA appears after summary is ready
- Recipients pre-filled from Zoom attendee list (names + emails)
- Phone-only attendees show number with blank name field
- Draft body includes summary and action items formatted for email
- mailto: deep-link opens Outlook with pre-filled content

### History / Search
- All past calls appear in history, newest first
- Full-text search across summaries and action items works
- Calls in "processing" state show spinner, not empty
- Calls with extraction errors show "needs review" state

## Edge Cases

### Zoom recording not found
- Cloud recording not enabled on banker's license → clear error UI + link to enable
- Recording webhook received but VTT not yet ready → retry once after 30s
- VTT malformed / empty → graceful degradation, show transcript unavailable message
- Zoom OAuth token expired mid-session → silent refresh, no UX interruption
- Zoom refresh token expired (90-day inactivity) → re-authentication prompt

### Gemini errors
- Gemini timeout → one auto-retry, then partial save with `needs_review` flag
- Gemini returns valid JSON missing action_items key → save empty array, save summary
- Gemini returns non-JSON → log error, flag call as `needs_review`, show summary unavailable
- Transcript too long for Gemini context window → truncate to last N tokens, flag as truncated

### Auth edge cases
- Banker accesses app without bank SSO session → redirect to SSO login
- Banker has no Zoom account connected → show "Connect Zoom" onboarding step
- Calendar not connected → dashboard shows manual entry, no error state

### Data / security
- Zoom webhook with invalid HMAC signature → 401, no processing, logged
- Webhook replay attack (duplicate webhook ID) → idempotent processing, no duplicate calls
- KMS key unavailable → backend returns 503, graceful error to frontend
- Cloud SQL connection exhausted (Cloud Run scaling) → connection pooling handles it

## Critical Paths (must work end-to-end before ship)

1. **Zoom OAuth → recording → summary → action items** — the entire V1 value prop in one flow
2. **Consent dialog → first recording → summary** — first-time user onboarding
3. **Webhook HMAC verification** — security gate; must reject invalid signatures
4. **Graceful Gemini failure → partial summary shown** — no banker should see a crash
5. **Draft follow-up email** — the primary CTA after every call

## Unit Test Targets (backend)

- `parseVTT(rawVTT)` → returns `{ text, speakers[] }` for valid VTT; handles malformed input
- `encryptToken(token)` / `decryptToken(ciphertext)` → KMS round-trip
- `verifyZoomWebhookSignature(payload, signature, secret)` → HMAC verify
- `validateGeminiResponse(json)` → Zod schema check; returns partial on missing fields
- `buildEmailDraft(summary, actionItems, attendees)` → correct mailto format
- `buildGeminiPrompt(vttText)` → prompt format, no injection surface

## Integration Test Targets (backend)

- `POST /api/zoom/webhook` with valid signature → processes and writes to Cloud SQL
- `POST /api/zoom/webhook` with invalid signature → 401, no DB write
- `GET /api/calls/:id/stream` → SSE delivers `done` event after processing completes
- `PATCH /api/action-items/:id` → updates DB, returns updated item
- Zoom token refresh flow → new access token written to DB (encrypted)

## E2E Test Targets (browser)

- Full Zoom OAuth connect flow (mock Zoom)
- Record → wait → summary appears via SSE
- Edit action item inline
- Click "Draft Follow-up Email" → mailto: opens with correct content
- History search returns correct results
